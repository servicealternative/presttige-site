# Presttige Production vs Legacy Map

## 1. Production (Active Pages)

The following pages are clearly part of the current live funnel.

### `index.html`
- Role in system:
  - Main landing page and entry point for the current private access flow
- Why it is considered production:
  - Contains the live first-step submission to the deployed `create-lead` API endpoint
  - Redirects users into the active funnel path
  - Uses current production analytics and production backend URLs
- Part of funnel:
  - Landing / Request Access
  - `create-lead` submission

### `check-email.html`
- Role in system:
  - Post-submission page instructing the user to check email for the next step
- Why it is considered production:
  - It is the direct redirect target after successful lead creation from `index.html`
  - It is part of the visible live user journey
- Part of funnel:
  - `check-email`

### `verify-email.html`
- Role in system:
  - Verification bridge page that forwards the browser to the deployed email-verification backend endpoint
- Why it is considered production:
  - Uses a real production API Gateway URL
  - Is directly connected to the current live token-verification flow
  - Its inline comments describe the live redirect behavior into `access-form.html`
- Part of funnel:
  - `verify-email`

### `access-form.html`
- Role in system:
  - Second-step application form for the access request process
- Why it is considered production:
  - Uses a real production `submit-access` backend endpoint
  - Reads the `lead_id` generated by the current live flow
  - Redirects to the current success page after submission
- Part of funnel:
  - `access-form`
  - `submit-access`

### `thank-you.html`
- Role in system:
  - Final confirmation page after successful application submission
- Why it is considered production:
  - It is the direct post-success destination from `access-form.html`
  - It completes the currently observed live frontend funnel
- Part of funnel:
  - `thank-you`

## 2. Likely Production (Needs Confirmation)

The following pages appear to be used in production, but their exact status still requires confirmation.

### `confirm.html`
- Function:
  - Founder payment confirmation page
  - Displays submitted name and email
  - Sends the user to a live Lambda Function URL to continue payment
- Explanation of uncertainty:
  - It is clearly wired to a real backend URL and looks production-ready
  - However, it belongs to a separate payment/founder flow, not the main access funnel
  - The repository alone does not confirm how actively this page is used today
- Risk if incorrectly classified:
  - If treated as legacy when still live, payment flow users could break
  - If modified prematurely, founder checkout behavior could be disrupted without backend visibility

### `cancel.html`
- Function:
  - Payment cancellation page for the founder/payment flow
  - Provides a retry route back into a live Lambda Function URL
- Explanation of uncertainty:
  - It is tied to the same live founder/payment backend family as `confirm.html`
  - It appears valid and usable, but its real production traffic level is not confirmed from the repository alone
- Risk if incorrectly classified:
  - If treated as non-production when still active, retry behavior after cancelled payments could fail
  - Incorrect changes could create a broken or dead-end payment recovery path
